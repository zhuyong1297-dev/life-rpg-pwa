# 项目长期记忆

## 架构决策

- `2026-07-19`：V2.0.0 采用手机优先、本地离线 PWA；IndexedDB 是唯一事实来源。
- `2026-07-19`：Obsidian 只保存版本设计、复盘归档和导出备份；滴答清单不再参与计分。
- `2026-07-19`：使用追加式 ledger event 表达奖励、撤销与兑换，保证每个数值可解释。
- `2026-07-19`：JSON 是完整备份与恢复格式；Markdown 只用于阅读和归档。
- `2026-07-19`：公开仓库只包含通用应用代码和人物资产，个人迁移数据永不提交。

## 产品取舍

- 14 天试运行只关注最多 3 项关键行为的坚持率，不用 XP 总量衡量成效。
- 游戏反馈服务于现实行动；升级只改变人物形象和反馈，不改变奖励倍率。
- `2026-07-19`：习惯目标明确区分次数和时长；时长型习惯保存实际分钟，达到目标才完成，但不按分钟放大奖励。
- `2026-07-19`：V2.1.0 新习惯可使用时间或次数三层目标；基础、标准、突破获得难度 XP 上限的 `60%/80%/100%`，任意层获得完整金币，同日升级只补 XP 差额。
- `2026-07-19`：三层目标在创建时录入严格递增阈值，完成时只选择层次；完成记录保存阈值和单位快照，后续调整不改写历史。
- `2026-07-19`：V2.2.0 习惯支持完整编辑；活动定义使用递增版本，完成保存名称、属性、难度和目标快照，历史奖励永不因编辑重算。
- `2026-07-19`：删除习惯采用可恢复归档，保留完成、流水和复盘；恢复后默认启用但不自动成为关键行为。
- `2026-07-19`：10 秒反馈层撤销继续保留；已完成状态另提供仅限今天、需要二次确认的持久取消，并用 correction 抵消全部关联奖励。
- `2026-07-19`：V2.3.0 三层时间目标统一按秒保存并保留秒/分钟输入偏好；组合目标支持每次固定时长和累计总量。
- `2026-07-19`：组合层次要求次数和时间不下降且至少一项增加；完成仍只选层次，奖励不随工作量放大。
- `2026-07-19`：V2.4.0 采用“现代冒险日志”的任务委托板界面；关键行动使用独立委托条，普通活动保持紧凑列表，功能名称始终优先于游戏化副标题。
- `2026-07-19`：V2.5.0 手机端改为五栏底部导航，中央 48px 创建按钮完全位于导航栏内；删除内容区固定悬浮按钮，从结构上避免遮挡任务完成按钮。
- `2026-07-19`：完成反馈先展开约 1.4 秒，再收缩为避开任务右侧操作区的 10 秒撤销条；减少动态效果时禁用非必要动画。
- `2026-07-19`：反馈层不截获页面点击，只有撤销按钮接收指针事件；弹窗层级高于反馈层，避免连续记录行动时被上一条反馈阻塞。
- `2026-07-19`：角色页阶段标签不得覆盖人物；430px 以下保持人物/阶段与等级/金币双列，并用视觉测试断言阶段、人物、等级和金币互不重叠。
- `2026-07-19`：V2.5.0 为每个新等级固化成长报告、称号和下一等级属性方向；Lv.3/6/10 与 Lv.15 起每 5 级提供不扣金币的阶段礼券，等级不影响计分倍率。
- `2026-07-19`：等级系统首次启用以当前等级为基线，10 秒撤销窗口结束且净 XP 仍达标后才固化；礼券领取与零数值 milestone 流水在同一事务写入。
- `2026-07-19`：完成声音必须在用户点击同步阶段创建或恢复单个 Web Audio `AudioContext`，奖励事务成功后播放；开启开关立即试听，后台返回时恢复上下文。
- `2026-07-19`：V2.5.0 反馈统一为柔和、清晰、强烈三档；普通完成、层次升级、角色升级分别使用 2/3/4 音与不同振动节奏，旧设置默认使用清晰档。
- `2026-07-19`：角色页不常驻完整等级路线或历史流水；等级路线、按月旅程档案和奖励目录改为按需弹层，角色页只保留 88px 路线摘要、最近 7 天最多 3 条记录和约 104px 商店摘要。
- `2026-07-19`：旅程章节全部从追加式流水派生，按月合并同一完成的层次升级，默认加载 3 个月并每次追加 6 个月，不新增汇总表。
- `2026-07-19`：奖励商品采用新增、编辑、可恢复停用而非物理删除；当前目标最多一个，完成反馈显示目标金币差距，商品修改不重写历史兑换。
- 通知和振动只是完成后的附加反馈，失败不得影响奖励写入。
- 不实现账号、同步、后台提醒、AI 建议、排行榜或社交功能。
- `2026-07-20`：V2.6.0 将业务日期固定为设备本地 `04:00` 分界；`occurredOn` 使用游戏日，`createdAt` 与 `exportedAt` 保留真实 ISO 时间。首次在凌晨四点前升级时延迟到当日 04:00 启用，历史日期永不重写。
- `2026-07-20`：分层习惯支持两层或三层；两层为基础/标准和 `60%/100%` XP，三层继续为 `60%/80%/100%` XP，旧目标与完成快照不自动修改。
- `2026-07-20`：一次性任务只在完成游戏日保留，下一次 04:00 后退出今天和关键名额；原活动与完成不删除，设置页使用折叠的已完成任务区查看。
- `2026-07-20`：行动日志由 active completion、未抵消 reward 和等级里程碑派生，使用月历加当月时间线；撤销完成、原奖励和 correction 不进入用户视图，完整审计仍保留。
