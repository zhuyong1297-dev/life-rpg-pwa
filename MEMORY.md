# 项目长期记忆

## 架构决策

- `2026-07-19`：V2.0.0 采用手机优先、本地离线 PWA；IndexedDB 是唯一事实来源。
- `2026-07-19`：Obsidian 只保存版本设计、复盘归档和导出备份；滴答清单不再参与计分。
- `2026-07-19`：使用追加式 ledger event 表达奖励、撤销与兑换，保证每个数值可解释。
- `2026-07-19`：JSON 是完整备份与恢复格式；Markdown 只用于阅读和归档。
- `2026-07-19`：公开仓库只包含通用应用代码和人物资产，个人迁移数据永不提交。

## 产品取舍

- 14 天试运行只关注最多 3 项关键行为的坚持率，不用 XP 总量衡量成效。
- 游戏反馈服务于现实行动；升级只改变人物形象和反馈，不改变奖励倍率。
- `2026-07-19`：习惯目标明确区分次数和时长；时长型习惯保存实际分钟，达到目标才完成，但不按分钟放大奖励。
- `2026-07-19`：V2.1.0 新习惯可使用时间或次数三层目标；基础、标准、突破获得难度 XP 上限的 `60%/80%/100%`，任意层获得完整金币，同日升级只补 XP 差额。
- `2026-07-19`：三层目标在创建时录入严格递增阈值，完成时只选择层次；完成记录保存阈值和单位快照，后续调整不改写历史。
- 通知和振动只是完成后的附加反馈，失败不得影响奖励写入。
- 不实现账号、同步、后台提醒、AI 建议、排行榜或社交功能。
