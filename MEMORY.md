# 项目长期记忆

- `2026-07-21`：V4.2.0 保持今天、角色、复盘、设置四个一级页面和中央创建按钮；新功能使用原生 URL Hash 全屏二级页面，目标规划器路径为 `#/coach/plan`，不增加 React Router 或底部导航项。
- `2026-07-21`：目标规划器用现实结果、行动链、最低标准、现实检查四步把目标转为 1 至 3 项行为；模板只预填结构，领域、难度、频率、触发条件和目标必须由用户确认。
- `2026-07-21`：唯一 `coachPlanDraft` 保存在现有 `settings` 表并进入 JSON schema 9 备份。存在当前赛季时只保存 `ready` 的下赛季方案，不创建活动或改写当前赛季。
- `2026-07-21`：规划草稿启用在一个 Dexie 事务中重新校验复用活动、创建新活动、切换关键行为、创建赛季并删除草稿；`Season.sourcePlanId` 与确定性 ID 保证双击幂等，失效复用活动使整笔事务回滚。
- `2026-07-21`：V4.0.0 将角色分类整体替换为六个现实成长领域，稳定 ID 为 `health`、`learning`、`creation`、`career`、`life`、`mindset`；中文文案可独立调整，数据无需再次迁移。
- `2026-07-21`：成长领域按行动最终改善的现实结果选择，专注和坚持只属于执行方式；旧属性仅保留在旧历史中，新领域 XP 从启用时的 0 开始，角色总 XP、金币和追加式流水不变。
- `2026-07-21`：V4 首次启动使用阻塞式逐项确认向导，建议只按旧属性映射且必须由用户明确确认；活动、当前赛季快照和启用状态在一个 Dexie 事务中迁移，已结束赛季不改写。
- `2026-07-21`：迁移长列表不能只显示已确认数量；未完成时必须显示首个未确认活动名称、为单项标记“待确认”并提供一键定位，避免用户在手机上被阻塞。
- `2026-07-21`：Dexie 升级到 version 3 但仍保持七张表；JSON 升级为 schema 7 并兼容 schema 1 至 6，恢复旧备份后重新进入领域迁移。
- `2026-07-21`：V4.1.0 为赛季增加一次性前三天校准。稳定生活蓝图由晨间唤醒、单点开工、夜间收尾组成；校准在单个 Dexie 事务中保存旧赛季定义、降级旧关键行为、创建新活动并把当前游戏日重设为第 1 天，重复提交不得重复创建。
- `2026-07-21`：活动可保存 `cue` 和 `protocol` 解释触发条件与执行协议；今天页默认折叠这些说明，执行提示不改变完成、XP 或金币规则。
- `2026-07-21`：赛季每日现实状态仅记录起床窗口是否达标、晨间精力和生活掌控感，同一游戏日可覆盖更新且不计分；赛季证据使用最近 7 条有效日期数据和三项行为的唯一完成日。
- `2026-07-21`：V4.1.0 备份使用 schema 8 并兼容 schema 1 至 7；IndexedDB 仍是 Dexie version 3 和七张表，校准及状态数据保存在现有 `seasons` 表中。
- `2026-07-21`：Dexie 中既有对象不会自动应用 Zod `.default()`；V4.1 首次预览曾因 schema 7 赛季缺少 `dailySignals` 导致旧设备启动空白。所有新增兼容字段必须在 `getSnapshot` 和直接写事务中先解析规范化，并用真实旧记录重载页面验证。
- `2026-07-21`：V4.1.0 正式发布保留稳定生活校准作为可选工具，但升级、初始化和读取永远不得调用它；用户选择自行添加行为，不把预览数据或推荐蓝图自动写入正式数据库。

- `2026-07-19`：V2.0.0 采用手机优先、本地离线 PWA；IndexedDB 是唯一事实来源。
- `2026-07-19`：Obsidian 只保存版本设计、复盘归档和导出备份；滴答清单不再参与计分。
- `2026-07-19`：使用追加式 ledger event 表达奖励、撤销与兑换，保证每个数值可解释。
- `2026-07-19`：JSON 是完整备份与恢复格式；Markdown 只用于阅读和归档。
- `2026-07-19`：公开仓库只包含通用应用代码和人物资产，个人迁移数据永不提交。

## 产品取舍

- V2 初始 14 天试运行只关注最多 3 项关键行为的坚持率；V3 改用两个 28 天赛季验证现实结果，两个阶段都不用 XP 总量衡量成效。
- 游戏反馈服务于现实行动；升级只改变人物形象和反馈，不改变奖励倍率。
- `2026-07-19`：习惯目标明确区分次数和时长；时长型习惯保存实际分钟，达到目标才完成，但不按分钟放大奖励。
- `2026-07-19`：V2.1.0 新习惯可使用时间或次数三层目标；基础、标准、突破获得难度 XP 上限的 `60%/80%/100%`，任意层获得完整金币，同日升级只补 XP 差额。
- `2026-07-19`：三层目标在创建时录入严格递增阈值，完成时只选择层次；完成记录保存阈值和单位快照，后续调整不改写历史。
- `2026-07-19`：V2.2.0 习惯支持完整编辑；活动定义使用递增版本，完成保存名称、属性、难度和目标快照，历史奖励永不因编辑重算。
- `2026-07-19`：删除习惯采用可恢复归档，保留完成、流水和复盘；恢复后默认启用但不自动成为关键行为。
- `2026-07-19`：10 秒反馈层撤销继续保留；已完成状态另提供仅限今天、需要二次确认的持久取消，并用 correction 抵消全部关联奖励。
- `2026-07-19`：V2.3.0 三层时间目标统一按秒保存并保留秒/分钟输入偏好；组合目标支持每次固定时长和累计总量。
- `2026-07-19`：组合层次要求次数和时间不下降且至少一项增加；完成仍只选层次，奖励不随工作量放大。
- `2026-07-19`：V2.4.0 采用“现代冒险日志”的任务委托板界面；关键行动使用独立委托条，普通活动保持紧凑列表，功能名称始终优先于游戏化副标题。
