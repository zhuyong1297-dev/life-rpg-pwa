# 项目协作规则

## 项目定位

这是“地球 Online”的即时反馈 PWA。系统以帮助用户持续完成少量关键行为为目标，游戏化反馈只用于强化现实行动，不以最大化任务数、经验值或金币为目标。

## 技术边界

- 技术栈：React、TypeScript、Vite、Dexie、Zod、Lucide、Vitest、Playwright、普通 CSS。
- IndexedDB 是运行数据的唯一事实来源；不增加后端、账号、云同步或状态管理框架。
- PWA 必须离线可用；正式路径为 `/life-rpg-pwa/`，预览路径为 `/life-rpg-pwa/preview/`。
- `main` 是正式版来源，`ui-redesign` 是预览版来源；预览构建必须使用独立 IndexedDB、manifest 和 Service Worker scope，不得读取或修改正式数据。
- 所有完成、撤销、兑换都使用追加式账本事件；不得直接修改历史事件。
- 数据导入必须先完整校验，再在单个事务中整体替换。

## 隐私红线

- 公开源码、构建产物和 Git 历史不得包含个人活动名称、账本、迁移数据、外部服务 ID、凭据信息或私有归档内容。
- 私有迁移和备份文件必须留在 Git 忽略目录或 Obsidian 私有库中。
- 不记录任何 secret 值；仅可说明凭据由 GitHub/Codex 本机环境管理。

## 产品规则

- 活跃关键行为最多 3 项。
- 简单、普通、困难、Boss 奖励分别为 `5/2`、`10/5`、`20/10`、`50/25`（XP/金币）。
- 习惯可使用单次完成或时间/次数分层目标；分层目标可选两层或三层，完成时只选择层次，不填写实际数值。
- 两层习惯使用基础/标准与 `60%/100%` XP；三层使用基础/标准/突破与 `60%/80%/100%` XP。任意首次完成发完整金币，升级只补 XP 差额。
- 分层时间目标可使用秒或分钟；组合目标支持“次数 × 每次时长”和“总次数 + 累计总时长”，时间统一按秒保存。
- 组合目标完成仍只选择层次；次数和时间不参与奖励倍率计算。
- 等级称号依次为启程者、行动者、践行者、塑造者；每级固化成长报告和里程碑，但不提高奖励倍率。
- Lv.3、Lv.6、Lv.10 以及 Lv.15 起每 5 级提供阶段礼券；礼券不扣金币、不可折现且每个节点只能领取一次。
- 奖励商品可新增、编辑和可恢复停用；同一时间最多一个当前奖励目标，商品修改不得改写历史兑换流水。
- 新等级只有在 10 秒撤销窗口结束且净 XP 仍满足时固化；首次启用以当前等级为基线，不补发历史里程碑。
- 旧时长习惯继续校验实际分钟，只有用户手动转换后才使用分层规则。
- 困难备注可选；一次性 Boss 必须填写 1 至 140 字成果，分层 Boss 习惯以层次选择作为完成证据。
- 习惯每天最多一个有效完成；任务只能有一个有效完成。
- 撤销追加 correction 事件；撤销后可重新完成，但同一活动同一天的净奖励最多一份。
- 习惯完整编辑递增活动版本且不重算历史；完成记录保留当时的名称、属性、难度和目标快照。
- 删除习惯实际执行可恢复归档；不得级联删除完成、流水或复盘。
- 已完成状态允许二次确认后取消今天的完成；昨天及更早记录不得从界面取消。
- 游戏日固定以设备本地时间 `04:00` 分界；所有每日限制、周统计、任务到期和业务流水统一使用游戏日，真实 ISO 时间戳不偏移。
- 一次性任务完成后保留到当前游戏日结束，下一游戏日从今天和关键名额中移除，但不删除活动、完成或流水。
- 不提供连击、漏签惩罚、经验扣除、金币衰减、排行榜或社交功能。

## 开发与验证

常用命令：

```bash
pnpm dev
pnpm test
pnpm test:e2e
pnpm build
pnpm privacy:scan
```

- 改动领域规则时必须增加或更新单元测试。
- 改动 IndexedDB 事务时必须覆盖幂等、回滚、撤销后重做。
- 改动界面时必须检查 Android 窄屏和桌面布局，并确认无溢出、遮挡或空白图片。
- `pnpm test:e2e` 固定验证正式基础路径；如果刚运行过 `pnpm build:preview`，必须先重新运行 `pnpm build`，避免 `dist` 路径不匹配造成空白页误报。
- 发布前依次通过测试、构建、隐私扫描和离线验收。

## 界面规则

- 视觉方向为“现代冒险日志”：成熟效率工具优先，RPG 词汇只作副标题和反馈氛围。
- 使用亮色、中性多色设计系统；主操作为森林绿，奖励为琥珀金，信息为蓝色，危险操作为珊瑚红。
- 圆角不超过 8px，操作触控区域至少 44px；不增加字体、动画或状态管理依赖。
- 今天页优先显示最多三项关键行动；手机端使用含中央创建按钮的五栏底部导航，禁止固定在内容区上的悬浮创建按钮；桌面端使用侧栏及双栏布局。
- 完成反馈先展开显示成长信息，再收缩成 10 秒撤销条；必须支持 `prefers-reduced-motion`。
- 声音与振动共享柔和、清晰、强烈三档；普通完成、层次升级和角色升级必须具有可辨别的 2/3/4 级语义反馈。
- 完成音效使用单个可复用 Web Audio `AudioContext`；开启开关必须先试听，完成事务成功后才能播放。
- 角色页只常驻紧凑等级摘要、最近 7 天最多 3 条有效轨迹和商店摘要；完整路线、月历行动日志与商品目录使用按需弹层。
- 行动日志只展示最终有效完成和正向里程碑；撤销记录及 correction 只保留在原始账本、JSON 和 Markdown 审计中。
- 创建表单使用渐进展开，默认只显示高频字段；属性、难度和组合目标按需展开。

## 文档规范

- 项目文档以简体中文为主，必要时保留技术英文术语。
- `MEMORY.md` 只记录长期有效的设计决策、踩坑和非机密运维信息。
- 学到长期有效的信息时更新 `MEMORY.md`，最终汇报时说明是否更新。
