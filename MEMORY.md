# 地球 Online 长期记忆

> 本文件记录当前有效且无法从代码直接看出的事实，不记录版本流水。发布历史由 Git、GitHub Releases 和 Obsidian `Version.md` 保存；新决策取代旧决策时直接更新原条目。

## 当前基线

- 正式版与预览版均运行 `V5.0.2`；两者复用同一 App 数据控制器，分别只读写 `earth-online-v2` 与 `earth-online-preview-v2`。
- 当前数据契约为 Dexie version 4、八张表、JSON schema 11；schema 1 至 10 可恢复，Markdown 只用于阅读归档。
- 正式入口为 `https://zhuyong1297-dev.github.io/life-rpg-pwa/`，预览入口为其 `/preview/` 子路径；两者使用独立数据库、manifest 和 Service Worker scope。
- `main` 与 `ui-redesign` 分别承载正式和预览；功能先在手机预览验收，再快进正式分支并创建不可变 Release。

## 架构与产品决策

- 完成、撤销、兑换和里程碑使用追加式流水；编辑、归档或删除活动不得重算或级联删除历史，完成与复盘保存必要快照。
- 数据导入必须先通过 Zod 完整校验，再在单个 Dexie 事务中整体替换；跨表写入必须原子、幂等并可回滚。
- 游戏日固定以设备本地 `04:00` 分界，历史 `occurredOn` 不重写；一次性任务只在完成游戏日保留于今天。
- 六个成长领域使用稳定 ID `health`、`learning`、`creation`、`career`、`life`、`mindset`；按现实结果分类，旧属性只留在旧历史。
- 活跃关键行为最多三项。28 天赛季保存目标、现实成功标准、起点、期望结果和核心活动快照；XP 不判断赛季成败。
- 本地建议必须展示依据和作用，未经确认不修改活动。目标规划器使用 `#/coach/plan`，当前赛季存在时只保存下赛季方案。
- 赛季校准是前三个游戏日的可选事务，只能由用户连续两次确认触发；升级、初始化、快照读取和 Service Worker 更新不得自动执行。
- 两层目标使用 `60%/100%` XP，三层使用 `60%/80%/100%` XP；任意首次达层发完整金币，同周期升级只补 XP。
- 每周次数与组合目标可逐次累计；普通进度不写零值流水，跨层只补净奖励，首条进度冻结本周规则且只能撤销最后一条有效进度。
- 今天页按关键行动、每日行动、本周进度、一次性任务排列；关键周行动在关键区执行、本周区展示，两处共用 completion。
- 五个一级入口为行动、成长、复盘、愿望、我的；新增复杂功能使用 Hash 二级页。界面保持紧凑，历史和管理列表按需打开。
- V5 以“行动、成长”为新高频界面，复盘、愿望和我的暂时复用既有真实功能；每日工作台按 `04:00` 游戏日排序时间行动，无时间行动允许仅当日优先，首页最多显示 5 项每日和 3 项每周行动。分层行动达到最低层即计入达标，但未到最高层时仍留在可行动区并提供直接升级，升级判断以 completion 快照为准。完成反馈先展示奖励再收缩为撤销或现实状态入口；成长页主卡优先突出旅者与等级进度，累计数值和重点领域保持轻量。Service Worker 更新取得控制权后，已受旧版本控制的页面自动刷新一次。
- 奖励系统以一个主愿望、候选队列和奖励券兑现现实愿望；每个游戏月补充 400 元奖励基金，最高结转 1200 元。金币锁定可退款，等级礼券在兑现后才正式领取。
- 等级、礼券、奖励愿望、声音和振动只增强反馈，不提高奖励倍率；不实现连击惩罚、排行榜、社交、云端 AI 或行为追踪。

## 关键踩坑与用户纠正

- Dexie 已有对象不会自动获得 Zod 默认值；新增兼容字段时，快照读取和直接写事务都必须先用当前 schema 规范化，并覆盖真实旧记录启动测试。
- 历史完成 ID 只能过滤一次性任务；判断跨日退出今天时必须同时检查 `activity.type === 'task'`，习惯不能被历史完成隐藏。
- 迁移向导不能只显示确认总数；必须显示首个未确认活动、单项待确认状态和定位入口。
- 中等宽度标题栏不能把标题夹在两个固定宽度操作之间；操作置于第一行，标题说明独占第二行，并保留约 500px 回归测试。
- 用户要求六个成长领域在手机和桌面均保持两列三行；320px 只收紧间距，不退化为单列。
- 用户偏好明确功能名、少量关键行动、人物与进度优先和按需展开；避免持续增长的常驻列表、嵌套卡片、成功消息使用警告样式和内容区悬浮按钮。
- 完成音频必须在用户手势中恢复单例 AudioContext，事务成功后播放；反馈 API 失败不能阻断奖励。
- 已安装 PWA 更新前先导出 JSON；不得通过卸载应用或清除站点数据处理缓存问题。

## 隐私与运维

- 公开仓库为 `zhuyong1297-dev/life-rpg-pwa`；私有迁移和备份只留在 Git 忽略目录或 Obsidian 私有库。
- GitHub 凭据由本机 GitHub CLI 管理，项目中不保存 secret。
- GitHub Pages 工作流同时构建正式与预览；两分支连续推送时，较早的同并发部署可能被正常取消。
- Git HTTPS 在当前网络中经常超时；发布默认运行 `pnpm publish:api -- <提交> <分支...>`，用 Git Data API 非强制快进并校验 tree SHA。普通 `git push` 仅作备用，凭据继续由 GitHub CLI 管理。
